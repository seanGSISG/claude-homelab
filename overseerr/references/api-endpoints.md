# Overseerr API Reference

**API Version:** 1.0.0
**Last Updated:** 2026-02-01

This is the documentation for the Overseerr API backend.

Two primary authentication methods are supported:

- **Cookie Authentication**: A valid sign-in to the `/auth/plex` or `/auth/local` will generate a valid authentication cookie.
- **API Key Authentication**: Sign-in is also possible by passing an `X-Api-Key` header along with a valid API Key generated by Overseerr.


**Base URL:** `{server}/api/v1`

## Authentication

### cookieAuth
**Type:** apiKey

Add API key as `connect.sid` in cookie.

```bash
-H "connect.sid: $API_KEY"
```

### apiKey
**Type:** apiKey

Add API key as `X-Api-Key` in header.

```bash
-H "X-Api-Key: $API_KEY"
```

## Quick Start

```bash
# Set environment variables
export BASE_URL="http://localhost:5055"
export API_KEY="your-api-key"

# Test connection
curl -s "$BASE_URL/api/v1/status" -H "X-Api-Key: $API_KEY"
```

## Endpoints by Category

### auth

#### GET /auth/me

Get logged-in user

Returns the currently logged-in user.

**Example Request:**
```bash
curl -X GET "$BASE_URL/auth/me" \
  -H "X-Api-Key: $API_KEY"
```
**Response Codes:**
- `200`: Object containing the logged-in user in JSON

---

#### POST /auth/plex

Sign in using a Plex token

Takes an `authToken` (Plex token) to log the user in. Generates a session cookie for use in further requests. If the user does not exist, and there are no other users, then a user will be created with full admin privileges. If a user logs in with access to the main Plex server, they will also have an account created, but without any permissions.

**Example Request:**
```bash
curl -X POST "$BASE_URL/auth/plex" \
  -H "X-Api-Key: $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"example": "data"}'
```
**Response Codes:**
- `200`: OK

---

#### POST /auth/local

Sign in using a local account

Takes an `email` and a `password` to log the user in. Generates a session cookie for use in further requests.

**Example Request:**
```bash
curl -X POST "$BASE_URL/auth/local" \
  -H "X-Api-Key: $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"example": "data"}'
```
**Response Codes:**
- `200`: OK

---

#### POST /auth/logout

Sign out and clear session cookie

Completely clear the session cookie and associated values, effectively signing the user out.

**Example Request:**
```bash
curl -X POST "$BASE_URL/auth/logout" \
  -H "X-Api-Key: $API_KEY" \
  -H "Content-Type: application/json"
```

**Response Codes:**
